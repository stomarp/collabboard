# CollabBoard Deployment Guide

CollabBoard is deployed as two services:

- Backend API and WebSocket server on Render
- Frontend Next.js app on Vercel

The backend must run on Render because the realtime layer depends on a long-running FastAPI process with WebSocket support.

## Production Architecture

```text
Vercel Next.js Frontend
  |
  | HTTPS REST requests
  | Secure WebSocket requests
  v

Render FastAPI Backend
  |
  | SQLAlchemy
  v

Render PostgreSQL

Render FastAPI Backend
  |
  | Redis Pub/Sub
  v

Render Key Value
```

## Backend: Render

Render uses the root-level `render.yaml`.

The Blueprint creates:

- `collabboard-api` web service
- `collabboard-db` PostgreSQL database
- `collabboard-redis` Render Key Value instance

The backend start script runs:

```bash
alembic upgrade head
uvicorn app.main:app --host 0.0.0.0 --port "$PORT"
```

## Render Environment Variables

Set these in Render after the Vercel frontend URL is available:

```env
FRONTEND_URL=https://your-collabboard-frontend.vercel.app
CORS_ORIGINS=https://your-collabboard-frontend.vercel.app
```

Render injects these from `render.yaml`:

```env
DATABASE_URL=<Render PostgreSQL connection string>
REDIS_URL=<Render Key Value connection string>
JWT_SECRET_KEY=<generated by Render>
```

## Frontend: Vercel

Create a Vercel project from the same GitHub repository.

Set the Vercel project root directory to:

```text
frontend
```

Set this Vercel environment variable:

```env
NEXT_PUBLIC_API_URL=https://your-collabboard-api.onrender.com
NEXT_PUBLIC_APP_NAME=CollabBoard
```

The frontend derives WebSocket URLs from `NEXT_PUBLIC_API_URL`.

Production behavior:

```text
https://your-collabboard-api.onrender.com
```

becomes:

```text
wss://your-collabboard-api.onrender.com/ws/boards/{boardId}?token=<JWT>
```

## Deployment Order

1. Merge deployment config to `main`.
2. Create the Render Blueprint from `render.yaml`.
3. Wait for backend, PostgreSQL, and Key Value services to deploy.
4. Confirm backend health:

```text
https://your-collabboard-api.onrender.com/health
```

5. Create the Vercel frontend project with root directory `frontend`.
6. Set `NEXT_PUBLIC_API_URL` in Vercel.
7. Deploy frontend.
8. Copy the Vercel frontend URL.
9. Update Render env vars:
   - `FRONTEND_URL`
   - `CORS_ORIGINS`
10. Redeploy the Render backend.
11. Test register, login, board creation, comments, drag/drop, and realtime sync.

## Production Smoke Test

After deployment:

- Open the Vercel frontend.
- Register a new account.
- Create a board.
- Create columns and tasks.
- Add and delete comments.
- Open the same board in two tabs.
- Confirm viewer count updates.
- Drag a task across columns.
- Confirm the second tab updates without refresh.

## Notes

Free Render services may cold start after inactivity. For a smoother recruiter demo, upgrade the backend web service before sharing widely.
